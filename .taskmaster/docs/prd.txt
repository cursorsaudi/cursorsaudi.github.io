# CursorSaudi.com — Product Requirements Document

## Overview

Build a community website for Cursor Saudi at cursorsaudi.com, inspired by CursorThailand.com. It is a single-page site (homepage only) with hero section, upcoming events, past events, and footer. Bilingual (Arabic default with RTL + English with LTR), dark mode only, deployed on GitHub Pages.

Reference projects:
- CursorThailand.com: Design/UX inspiration (hero, events, community feel)
- /Users/ma7dev/Projects/ma7.dev: Astro + Tailwind CSS v4 + GitHub Pages deployment reference

## Tech Stack

- Framework: Astro 5.x (static site generation)
- Styling: Tailwind CSS v4 via @tailwindcss/vite plugin
- Language: TypeScript (strict mode)
- Package manager: pnpm
- Deployment: GitHub Pages via GitHub Actions
- Domain: cursorsaudi.com (CNAME file in public/)

## Functional Requirements

### 1. Project Initialization & Configuration

Initialize an Astro 5.x project with TypeScript strict mode. Create:
- `package.json` with dependencies: astro ^5.x, @astrojs/sitemap, @tailwindcss/vite, tailwindcss v4, typescript
- `astro.config.ts` with site set to "https://cursorsaudi.com", integrations: @astrojs/sitemap, vite plugins: @tailwindcss/vite
- `tsconfig.json` extending astro/tsconfigs/strict with path alias @/* -> ./src/*
- `src/config.ts` as single source of truth for site metadata (title, description, author, social links for Luma and X/Twitter)

### 2. Tailwind CSS v4 & Global Styles

Configure Tailwind CSS v4 with dark-mode-only theme:
- Create `src/styles/global.css` with Tailwind imports and CSS custom properties
- Colors: background #0d1117, surface/cards #161b22 with 1px solid #30363d border, primary text #e6edf3, secondary text #8b949e, accent #58a6ff, accent hover #79c0ff
- Arabic font stack: "Noto Kufi Arabic", "IBM Plex Arabic", system-ui
- English font stack: Inter, system-ui, sans-serif
- Import Google Fonts (Noto Kufi Arabic, Inter) in the base layout

### 3. Internationalization (i18n) System

Build a client-side i18n system with Arabic as default:
- Create `src/i18n/ar.json` with Arabic UI strings (hero text, section headings, button labels, footer text)
- Create `src/i18n/en.json` with English UI strings
- Create `src/i18n/index.ts` with a `getTranslation(lang: "ar" | "en", key: string)` utility function
- Create `public/toggle-lang.js` that reads localStorage before page render to set `<html dir="rtl" lang="ar">` or `<html dir="ltr" lang="en">`, preventing layout flash (FOUC)
- Language preference stored in localStorage key "lang", defaults to "ar"

### 4. Event Content Collection

Define a Zod-validated content collection for events:
- Create `src/content.config.ts` defining the "events" collection
- Collection loader points to `src/data/events/` directory
- Frontmatter schema fields:
  - title (string, required) — English event name
  - titleAr (string, required) — Arabic event name
  - date (date, required) — event date
  - location (string, required) — English venue
  - locationAr (string, required) — Arabic venue
  - attendees (number, optional) — attendee count for past events
  - host (string, optional) — host name
  - lumaUrl (string, optional) — Luma registration link
  - description (string, required) — English description
  - descriptionAr (string, required) — Arabic description
  - isPast (boolean, required) — whether event has already happened
- Create `src/utils/events.ts` with stateless helper functions: getSortedEvents, getUpcomingEvents, getPastEvents
- Create 2 sample event Markdown files in `src/data/events/`

### 5. Base Layout

Create `src/layouts/Layout.astro`:
- HTML structure with dark mode class on html element
- Dynamic dir attribute (rtl/ltr) and lang attribute (ar/en) based on language
- Meta tags: charset, viewport, description, Open Graph (og:title, og:description, og:type, og:url)
- Favicon link
- Google Fonts import (Noto Kufi Arabic, Inter)
- Include toggle-lang.js as inline script in head (before render)
- Import global.css
- Slot for page content

### 6. Header Component

Create `src/components/Header.astro`:
- Site logo/brand: "Cursor Saudi | المجتمع" (or appropriate bilingual branding)
- Include LangToggle component
- Responsive design: works on mobile and desktop
- Sticky or fixed header (optional)

### 7. Language Toggle Component

Create `src/components/LangToggle.astro`:
- AR/EN toggle button
- Client-side script that toggles localStorage "lang" value and reloads/updates the page
- Visual indicator of current language

### 8. Hero Section Component

Create `src/components/Hero.astro`:
- Community tagline in both languages (shown based on current language)
- Brief description of the Cursor Saudi community
- CTA button linking to the next upcoming event (Luma registration) or scrolling to upcoming events section
- Visually prominent section with the dark theme accent colors

### 9. Event Card Component

Create `src/components/EventCard.astro`:
- Reusable card component accepting event data as props
- For upcoming events: show date, title (based on language), location, and "Register" link to Luma
- For past events: show date, title (based on language), location, attendee count, host name
- Card uses surface color #161b22 with border #30363d
- Responsive layout

### 10. Upcoming Events Section

Create `src/components/UpcomingEvents.astro`:
- Section heading (bilingual)
- Queries content collection for upcoming events (isPast === false), sorted by date ascending
- Renders EventCard for each upcoming event
- Shows a message if no upcoming events

### 11. Past Events Section

Create `src/components/PastEvents.astro`:
- Section heading (bilingual)
- Queries content collection for past events (isPast === true), sorted by date descending
- Renders EventCard for each past event
- Shows attendee count and host information

### 12. Footer Component

Create `src/components/Footer.astro`:
- Social links: Luma profile, X/Twitter profile
- Copyright text: "© 2026 Cursor Saudi" (bilingual)
- Minimal, clean design consistent with dark theme

### 13. Homepage Assembly

Create `src/pages/index.astro`:
- Uses Layout.astro as the base layout
- Assembles sections in order: Header, Hero, UpcomingEvents, PastEvents, Footer
- Passes site config data as needed

### 14. Static Assets & Public Files

Create files in public/:
- `public/.nojekyll` — empty file to bypass Jekyll processing on GitHub Pages
- `public/CNAME` — contains "cursorsaudi.com"
- `public/favicon.svg` — simple favicon (Cursor Saudi branding or placeholder)

### 15. GitHub Actions Deployment

Create `.github/workflows/deploy.yml`:
- Trigger: push to main branch + manual workflow_dispatch
- Permissions: contents read, pages write, id-token write
- Concurrency: github-pages group with cancel-in-progress
- Build job on ubuntu-latest:
  1. Checkout code
  2. Setup pnpm v9
  3. Setup Node.js v20 with pnpm cache
  4. Install dependencies: pnpm install --frozen-lockfile
  5. Build: pnpm run build
  6. Upload artifact: actions/upload-pages-artifact@v3 with path ./dist
- Deploy job depending on build:
  1. Deploy to GitHub Pages: actions/deploy-pages@v4

### 16. Final Verification & Polish

- Verify the site builds successfully with `pnpm run build`
- Test RTL (Arabic) and LTR (English) layouts both render correctly
- Verify event content collection loads and displays sample events
- Ensure the language toggle works (switches dir, lang, and displayed text)
- Verify the GitHub Actions workflow is valid YAML
- Check responsive design on mobile viewport sizes

# Notion Event Data Import — Supplemental PRD

## Context

All 19 original tasks are complete. The project now has a working Astro site with events rendered from sample Markdown files. A Notion export was provided containing the real event data for Cursor Saudi — 17 events spanning May 2025 through November 2026.

The event content collection schema needs to be updated to match the actual Notion CSV fields, and all 17 events need to be converted from CSV into Markdown content files.

Source CSV: `ExportBlock-f616ed77-905b-4068-9702-1263e7bca7eb-Part-1/Cursor Community in 🇸🇦/Events 27d321ff1409804880b6de61fa1c6336.csv`

## Requirements

### 1. Update Event Content Collection Schema

Update `src/content.config.ts` to match the real Notion data fields:

**Add these new required/optional fields:**
- `type`: z.enum(["meetup", "hackathon", "workshop"]) — required, every event has a type
- `speakers`: z.array(z.string()).optional() — speaker names, some events have multiple
- `slides`: z.array(z.string()).optional() — slide URLs, some events have multiple URLs
- `place`: z.string().optional() — venue name (mostly empty in current data)

**Remove these fields:**
- `isPast` — no longer stored; computed at build time from `date < new Date()`
- `attendees` — not present in Notion data
- `host` — not present in Notion data

**Make these fields optional (Arabic translations not yet available):**
- `titleAr`: z.string().optional() — was required, make optional
- `locationAr`: z.string().optional() — was required, make optional
- `descriptionAr`: z.string().optional() — was required, make optional

Keep `title`, `date`, `location`, `description` as required. Keep `lumaUrl` as optional.

### 2. Update Event Utility Functions

Update `src/utils/events.ts`:
- Remove any filtering based on `isPast` frontmatter field
- `getUpcomingEvents()`: filter events where `date >= new Date()`, sort ascending by date
- `getPastEvents()`: filter events where `date < new Date()`, sort descending by date
- Both functions should use the current date at build time for comparison
